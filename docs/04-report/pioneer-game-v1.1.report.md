## 1. 개발 개요

### 1.1 계획 대비 실제 범위

v1.1 Plan에서 정의한 4개 핵심 기능의 구현 현황:

| 계획 기능 | 구현 상태 | 비고 |
|-----------|-----------|------|
| 저장/불러오기 (localStorage) | ⚠️ 보류 | v1.2 예정 — 먼저 게임플레이 심화 우선 |
| 시세 히스토리 차트 (Sparkline) | ⚠️ 보류 | v1.2 예정 |
| 항구 특산 자원 보너스 | ⚠️ 대체 구현 | RESOURCE_REGIONS 지역별 시세 차등으로 대체 |
| 날씨·해류 이벤트 | ✅ 초과 구현 | 전역 날씨 대신 배별 이벤트 시스템으로 고도화 |

### 1.2 초과 구현 기능 (사용자 요청 반영)

| 기능 | 구현 내용 |
|------|-----------|
| 함선 다양화 | 11종 (통통배~프리깃), 각 지역 특화 설명 포함 |
| 특수 선원 시스템 | 9명 특수 선원 풀, 레어리티 4등급, 지역별 navBonus/tradeBonus |
| 항구 확장 | 23개 항구, 5개 지역 (유럽/지중해/아라비아/남아시아/동아시아) |
| 부스터 시스템 | 연료 2배 소모, 속도 +43%, 연료 20% 미만 시 사용 불가 |
| 연료·선체 내구도 | fuel(0~100), hull(0~100), 승무원 repair 스탯 연동 |
| 최대 24배 줌 | 휠/핀치 확대, 항해 가까이 감상 가능 |
| 이벤트 시스템 | 6종 (난파선/폭풍/해적/고래/보물/해류), 5초 간격, 즉각 효과 + 클릭 보상 |
| 지수 세금 | `50×1.3^(lv-1)`, 다이아몬드 대납, 금 부족 시 경고 |
| 퀘스트 시스템 | 배달·방문·무역 3종, 최대 3개 수주, 자동 보상 지급 |
| 시세 지역 차등 | RESOURCE_REGIONS — 자원별 싸고 비싼 지역 구분 |
| 인트로 슬라이드 | 스토리 1장 (시스템 설명 제거, 인게임 튜토리얼로 이관) |
| 인게임 튜토리얼 | 4단계 (depart→sailing→sell→buy), 단계별 힌트 + 시장 버튼 하이라이트 |

## 2. 기술 구현 상세

### 2.1 아키텍처 (단일 파일 유지)

```
src/App.jsx (1,417줄)
│
├── 모듈 레벨 상수
│   ├── SHIP_TYPES (11종), PORTS (23개), RESOURCES (10종)
│   ├── PORT_SHIPS (항구별 구매 가능 함선)
│   ├── REGION_STYLE (5개 지역 스타일)
│   ├── RESOURCE_REGIONS (자원별 지역 시세)
│   ├── SPECIAL_CREW_POOL (9명 특수 선원)
│   └── PORT_INFO, calcTax, makePrediction, generateQuests
│
├── gsRef mirror 패턴
│   └── useRef + custom setGs wrapper (stale closure 방지)
│
├── State
│   ├── gs: GameState (gold, gems, ships[], crew[], quests 등)
│   ├── prices{}, mapEvents[], tutorialPhase
│   └── mapView (x, y, zoom — 최대 24×)
│
├── Effects (4개)
│   ├── 초기화 (가격 + 퀘스트 생성)
│   ├── 이동 루프 300ms (booster, storm, 방문 퀘스트)
│   ├── 이벤트 생성 5초 간격 (6종)
│   └── 시세 갱신 1시간 + 세금 600초
│
└── UI
    ├── 인트로 슬라이드 (1장 스토리)
    ├── 헤더 (금/세금/보석/승무원/퀘스트)
    ├── 인게임 튜토리얼 힌트 바 (4단계)
    ├── 지도 (24× 줌, pan, 이벤트, 항로)
    ├── 시장 팝업 (매입/매도)
    └── 함대 패널 (4탭: 정보/화물/승무원/업그레이드)
```

### 2.2 핵심 로직 설계 결정

| 항목 | 결정 | 이유 |
|------|------|------|
| 이벤트 → gsRef 접근 | `gsRef.current` 사용 | setInterval closure에서 최신 ships 접근 필요 |
| 이벤트 state 분리 | `mapEvents` 별도 state | gs를 오염시키지 않음, 순수 시각 요소 |
| 퀘스트 완료 감지 | setGs updater 내부 | 단일 트랜잭션으로 gold+완료 처리 |
| stormUntil timestamp | ship 필드에 직접 저장 | 개별 배 독립 적용, weather 전역 state 불필요 |
| 튜토리얼 상태 전환 | useEffect 체인 | atPort/isMoving 변화 감지로 자동 진행 |
