Design Review

Pioneer — UX & 튜토리얼 개선안

현재 온보딩·가이드의 미비점을 진단하고, 구체적인 개선 방향과 구현 스펙을 제안합니다.

목차

  1. 현황 진단 — 무엇이 문제인가

  3. 튜토리얼 재설계 — 3단계 점진적 공개

Section 01
현황 진단
코드와 기획서를 기반으로 현재 온보딩·UX의 강점과 약점을 분류했습니다.

튜토리얼 커버리지 심각

  • TUTORIAL_STEPS는 5단계(배 선택→출항→항해→판매→구매)뿐
  • 승무원·업그레이드·정보 구매·세금·배달 의뢰·일일 목표·날씨·지도 이벤트 등 핵심 10+ 시스템에 튜토리얼 없음
  • 튜토리얼 완료 후 'done' 상태에서 플레이어는 무엇을 해야 하는지 알 수 없음
  • 인트로 슬라이드 1장 — 세계관만 전달하고 플레이 방법은 설명 없음

시각적 유도 부재 심각

  • 튜토리얼 스텝이 텍스트 바 형태 → 어떤 UI 요소를 눌러야 하는지 시각적으로 가리키지 않음
  • 중요 버튼 하이라이트(스포트라이트 오버레이) 없음
  • 배 선택 → 목적지 선택 → 거래소 흐름이 맥락 없이 이어짐
  • 항구 이름만 있고 "어디가 싸고 비싼지" 초반에 시각적 단서 없음

정보 과부하 주의

  • 첫 화면에 함대 목록·HUD·조언 바·화물 미니창·지도 버튼·항구 라벨·이벤트 아이콘이 동시에 노출
  • 우측 패널 4개 탭(정보·함대·퀘스트·승무원)이 항상 접근 가능 → 초반 플레이어에게 의미 없는 노이즈
  • 항로 잠김 항구도 동일한 시각 비중 → 방문 가능한 항구가 강조되지 않음

빈 상태 & 오류 처리 주의

  • 첫 세션: 금 0, 선원 0~1, 화물 8개 — 거래소 열면 "판매 가능한 화물 없음" 빈 슬롯만 표시 가능성
  • 항구 해금 실패 시 로그 텍스트만 출력 (시각 피드백 없음)
  • 연료·내구도가 낮을 때 출항 경고 없음
  • 승무원 0명 출항 시도 → 로그 텍스트 출력 후 묻힘

잘 되고 있는 것 유지

  • 상시 조언 바: 맥락 기반 다음 행동 안내 — 방향성은 올바름
  • 항구 해금 라벨: 지도 위 누적 판매금 조건 표시
  • 거래소 분할 레이아웃: 차트·조작부·목록 분리
  • 가격 방향 힌트: 상품별 ▲▼ 변동률 표시
  • 항로 확정형 UX: 시세 확인 후 목적지 확정 버튼

첫 세션 목표 불명확 주의

  • "양털을 어디에서 팔면 좋은가"를 플레이어가 스스로 발견해야 함
  • 퀘스트·일일 목표가 처음부터 열려 있지만 화면 밖 탭에 숨어 있음
  • 최초 목표(양털 8개 판매)가 HUD나 지도에 시각적으로 표시되지 않음
  • 누적 판매금 진행 바(다음 지역 해금까지 얼마 남았는지) 없음
핵심 문제 요약: 튜토리얼이 "무엇을 클릭하라"를 알려주지만 "왜 그 행동을 해야 하는가"와 "다음에 뭘 해야 하는가"를 알려주지 않는다. 처음 플레이하는 사람은 5단계 튜토리얼 이후 방향을 잃는다.

Section 03
튜토리얼 재설계 — 3단계 점진적 공개
기존 5-스텝 선형 튜토리얼을 "필수 루프 → 확장 시스템 → 자유 플레이" 3단계로 재구성합니다.
설계 원칙: 각 단계는 플레이어가 직접 행동하고 즉각적인 보상을 받은 뒤 다음 단계가 열립니다. 설명보다 행동이 먼저입니다.
Phase 1 — 필수 루프 (약 3~5분)
1
인트로 슬라이드 개편 (3장) 개편
기존 1장(세계관 텍스트) → 3장으로 확장. 각 장은 10초 이내로 읽힌다.
①
슬라이드 1: 상황 설정 — "당신은 리스본의 초보 상인. 통통배 1척, 양털 8개가 전부입니다." 지도 미리보기 배경 포함.
②
슬라이드 2: 핵심 루프 도식 — "싼 곳에서 사서 → 비싼 곳에서 팔고 → 더 큰 배와 승무원을 산다" 3아이콘 흐름도.
③
슬라이드 3: 첫 목표 제시 — "양털은 런던·함부르크·앤트워프에서 비싸게 팔립니다. 항해해서 팔아보세요!" 지도 위 3개 항구를 명시적으로 하이라이트한 미니맵 이미지 포함.
2
배 선택 — 스포트라이트 오버레이 신규
함대 목록 또는 지도 위 배 아이콘에 펄싱 링 애니메이션 + 화살표 툴팁 표시.
①
반투명 오버레이로 배경 어둡게 처리. 함대 목록 영역과 지도 위 배 아이콘만 밝게 유지.
②
배 아이콘 주변에 pulse 링 애니메이션 (CSS @keyframes ring-pulse).
③
배 클릭 즉시 오버레이 해제 → 다음 스텝(목적지 선택) 자동 진입.
3
목적지 선택 — 가격 힌트 오버레이 신규
목적지 선택 모드 진입 시, 보유 화물(양털)이 비싼 항구를 지도 위에서 강조 표시.
①
항구 라벨 아래에 "양털 ▲ 비쌈" 배지를 런던·함부르크·앤트워프에 표시. (RESOURCE_REGIONS 데이터 활용)
②
해당 항구 아이콘에 금색 테두리 하이라이트 적용.
③
조언 바: "💡 양털은 유럽 항구에서 비쌉니다. 항구를 눌러 시세를 확인하세요." 텍스트 유지.
4
항해 중 — 미니 HUD 보조 개선
항해 시작 후 "도착까지 N% 완료" 진행 바와 현재 날씨·연료·내구도를 지도 하단에 컴팩트하게 표시.
①
지도 하단 화물 미니창 옆에 항해 진행 바 (선택된 배가 항해 중일 때만 노출).
②
튜토리얼 sailing 스텝 중: "📍 추적 버튼을 눌러 배를 따라가 보세요" 버튼을 툴팁으로 강조.
5
도착 & 첫 판매 — 완료 연출 신규
항구 도착 시 토스트 알림 + 시장 버튼 스포트라이트. 판매 완료 후 Phase 1 완료 팝업 표시.
①
도착 시 중앙 상단 토스트: "⚓ [항구명] 도착! 시장에서 화물을 판매하세요." + 시장 버튼 스포트라이트.
②
판매 완료 시: 골드 증가 애니메이션 (+XXX 금 플로팅 텍스트) + 토스트 알림.
③
Phase 1 완료 팝업: "첫 무역 성공! 이제 더 멀리 항해하려면 화물을 사야 합니다." + "Phase 2: 무역 루프 확장" 프리뷰.
Phase 2 — 확장 시스템 소개 (첫 판매 완료 이후)
시스템 트리거 조건 노출 방식
화물 매입 첫 판매 완료 후 즉시 시장 탭 "매입"에 깜빡이는 배지 + 조언 바 "이 항구에서 싼 화물을 사서 돌아가세요"
선원 고용 첫 항해 완료 (누적 판매금 500+) 우측 패널 선원 탭에 뱃지(!) + 첫 오픈 시 "선원을 고용하면 항해 속도와 거래 효율이 오릅니다" 1회 툴팁
정보 구매 누적 판매금 1,500+ (지중해 해금 조건 근접) 항구 시세창 하단에 "거리 소문(300금)" 버튼 강조 + 첫 클릭 시 정보 구매 설명 팝업
퀘스트 3번째 항해 완료 우측 패널 퀘스트 탭 뱃지(!) 추가 + "새 퀘스트가 생겼습니다" 토스트
일일 목표 첫 세금 납부 이후 일일 목표 버튼에 배지 + "오늘 목표를 달성하면 보석을 얻습니다" 1회 설명
세금 시스템 세금 납부 D-30분 전 HUD 세금 카운트다운 빨간색 전환 + 조언 바 "세금이 [N]분 뒤 징수됩니다. 보석으로 면제 가능합니다"
배 업그레이드 두 번째 배 구매 또는 누적 판매금 5,000+ 함대 탭 배 카드에 "업그레이드 가능" 배지 + 첫 오픈 시 설명
Phase 3 — 자유 플레이 (튜토리얼 완전 종료)

진입 조건 달성 목표

  • 조건: 누적 판매금 3,000금 달성 OR 3개 시스템(선원·퀘스트·화물 매입) 사용 경험
  • 튜토리얼 오버레이 완전 제거, 조언 바는 계속 동작 (상황별 맥락 힌트로 전환)
  • Phase 3 진입 시 1회 요약 팝업: "이제 당신은 진정한 상인입니다. 동아시아까지 항해해 보세요!" + 다음 지역(지중해) 해금 진행 바 표시
Section 04
UX 피드백 시스템
플레이어의 주요 행동 순간마다 즉각적인 시각 피드백이 필요합니다. 현재 대부분 로그 텍스트에만 의존하고 있습니다.
AS-IS (현재)
  • 판매 완료 → 로그에 텍스트 1줄
  • 출항 → 로그에 "항해 중..." 텍스트
  • 퀘스트 완료 → 로그 텍스트
  • 항구 해금 실패 → 로그에 묻힘
  • 연료 부족 출항 → 별도 경고 없음
  • 세금 임박 → 카운트다운만
TO-BE (개선 후)
  • 판매 완료 → 골드 +N 플로팅 텍스트 + 토스트
  • 출항 → 화면 중앙 상단 토스트 "항해 시작"
  • 퀘스트 완료 → 중앙 완료 팝업 + 보상 애니메이션
  • 해금 실패 → 지도 위 잠금 아이콘 흔들림 + 토스트
  • 저연료/저내구도 → 출항 전 경고 모달
  • 세금 임박 10분 → 화면 가장자리 빨간 테두리 펄스
트리거 이벤트 피드백 형태 위치 지속시간
화물 판매 완료 플로팅 텍스트 "+[N]금" + 토스트 "판매 완료" HUD 금화 위치에서 위로 떠오름 1.5초
항구 도착 상단 중앙 토스트 "[항구명] 도착! 시장 버튼 하이라이트" 화면 상단 중앙 4초
퀘스트 완료 중앙 완료 팝업 (보상 표시 + "완료!" 텍스트) 화면 중앙 클릭 시 닫힘
새 항구 발견 토스트 "새 항구 발견! [항구명] 항로 개통" + 지도 핀 이펙트 화면 상단 우측 3초

Section 06
구체적 구현 사항
모든 개선 사항을 App.jsx 중심으로 구현할 때 필요한 상태·컴포넌트·데이터를 명세합니다.

인트로 슬라이드 확장

  • INTRO_SLIDES 배열: 1개 → 3개
  • 슬라이드 2: type: 'loop-diagram' 필드 추가 → 아이콘 흐름도 렌더링
  • 슬라이드 3: highlightPorts: ['london','hamburg','antwerp'] → 미니맵 위 하이라이트
  • 슬라이드 자동 진행: autoAdvance: true, delay: 8000 (클릭으로 건너뛰기 가능)
  • 인트로 스킵 버튼: 2회차 이상 플레이어용

스포트라이트 오버레이

  • 상태: tutorialSpotlight: string | null (대상 요소 selector)
  • TutorialSpotlight 컴포넌트: 반투명 오버레이 + 구멍(clip-path) + 툴팁 화살표
  • 튜토리얼 스텝 → 스포트라이트 매핑:
    · select: '.fleet-list, .ship-icon-on-map'
    · depart: '.port-icon.highlight'
    · sell: '.market-button'
  • 스포트라이트 대상 클릭 시 자동 해제

토스트 알림 시스템

  • 상태: toasts: Array<{id, msg, type, duration}>
  • type: 'success' | 'warn' | 'info' | 'error'
  • addToast(msg, type, duration) 헬퍼 함수
  • 위치: 화면 상단 중앙, 최대 3개 스택, 오래된 것 아래
  • 애니메이션: 위에서 슬라이드인 → duration 후 페이드아웃
  • 기존 addLog에서 분기: 중요 이벤트는 토스트 + 로그 동시 기록

플로팅 텍스트 (+N금)

  • 상태: floatingTexts: Array<{id, text, x, y, color}>
  • addFloatingText(text, anchorEl, color) 헬퍼
  • 판매 완료 시 HUD 금화 위치에서 위로 이동하며 페이드아웃
  • CSS: @keyframes float-up { 0% translateY(0); 100% translateY(-40px) opacity 0 }
  • 색상: 금 판매 → #f59e0b, 보석 획득 → #38bdf8, 손실 → #f87171

누적 판매금 진행 바

  • 현재 지역 → 다음 해금 지역 계산: REGION_PORT_UNLOCK_GOLD_REQ 순서대로
  • 진행률: gs.totalEarned / nextMilestone * 100
  • HUD에 UnlockProgressBar 컴포넌트 추가
  • 마일스톤 달성 시: 토스트 + 지도에서 해당 지역 항구 language 파티클 이펙트
  • 공간 없을 때: 축약 모드 "지중해 83%" 텍스트만

가격 힌트 지도 오버레이

  • 조건: tutorialPhase === 'depart' AND routeMode === true
  • 보유 화물 목록에서 RESOURCE_REGIONS[res].expensive 지역 추출
  • 해당 지역 항구에 hintBadge 렌더링: "[화물명] ▲ 비쌈"
  • 배지는 항구 라벨 아래 별도 DOM 요소 (지도 좌표 변환 필요)
  • Phase 1 종료 후 자동 숨김 (showCargoHints: boolean 상태)

저연료/저내구도 경고

  • chooseDestinationPort 내에서 출항 전 체크
  • 연료 < 30% OR 내구도 < 30% → showDepartureWarn: boolean 상태 ON
  • 경고 모달: 현재 연료·내구도 표시 + "그래도 출항" / "취소" 버튼
  • "그래도 출항" 클릭 시 pendingDeparture 실행 후 모달 닫기

시스템 첫 노출 툴팁

  • 상태: seenSystems: Set<string> (localStorage 저장)
  • 체크 항목: 'crew' | 'portInfo' | 'tax' | 'quest' | 'upgrade' | 'delivery'
  • 각 탭/패널 첫 오픈 시 !seenSystems.has(key) → 1회 툴팁 표시
  • 툴팁 닫으면 seenSystems.add(key) + localStorage 업데이트
  • 툴팁 형태: 해당 UI 요소 옆에 나타나는 인라인 말풍선 (모달 X)
Phase 2 잠금 해제 트리거 정리 — 아래 상태를 gs 또는 별도 tutorialProgress 상태에 추가:

tutorialProgress: { firstSaleDone: bool, voyageCount: number, crewHired: bool, questOpened: bool, portInfoBought: bool }

각 트리거 달성 시 해당 시스템 배지(unreadBadge) ON + 조언 바 메시지 갱신.
