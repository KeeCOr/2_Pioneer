핵심 게임 루프

⚓
Core Value: 세금에 맞서 얼마나 오래 살아남는가. 생존이 목표, 방치가 플레이 방식.
1~2분
최소 세션 — 배 출발만 해도 의미 있음
5~10분
표준 세션 — 퀘스트 + 선원 관리
20~30분
풀 세션 — 배 업그레이드 + 이벤트 대응

루프 구조

접속
 └─ 도착한 배 확인 → 화물 판매
 └─ 퀘스트 처리 (완료/실패)
 └─ 이벤트 확인 (팝업 처리)
 └─ 다음 항구로 배 출발
     └─ [방치 — 앱 종료 가능]
         └─ 1시간마다 시세 변동
         └─ 배 도착 시 로그 기록
             └─ [재접속]

도박성 분류 회피 — 설계 근거

⚖️
비판: 불확실한 가격 변동이 확률 게임/베팅으로 인식될 수 있음 → 앱스토어 심의 거부 위험

핵심 구분: 경제 불확실성 vs 확률 베팅

앱스토어 가이드라인(Apple 4.3, Google Play 도박 정책)은 "결과가 무작위이며 실제 화폐로 베팅하는 행위"를 도박으로 정의한다. Pioneer의 가격 변동은 다음 이유로 이 정의에 해당하지 않는다.

구분도박 게임Pioneer
결과 결정 방식순수 난수 (RNG)수급 알고리즘 (결정론적)
플레이어 영향력없음루트 선택·타이밍·선원 구성으로 수익 차이 발생
실패 반복 가능성동일 확률 반복정보 구매·경험 축적으로 개선 가능
실제 화폐 베팅있음없음 (인앱 결제는 보석 구매, 게임 결과에 직접 베팅 아님)
💡
이는 Stardew Valley 작물 시세, Civilization 교역 시스템과 동일한 경제 시뮬레이션 범주다. 두 게임 모두 앱스토어에서 도박성 문제 없이 유통 중.

가격 변동 — 투명성 원칙

수급 알고리즘 공개 원칙
├── 각 자원의 현재 가격은 언제나 화면에 표시
├── 가격 히스토리(차트)를 통해 추세 파악 가능
├── "정보 구매" 시스템은 불확실성을 줄이는 수단 (완전 제거 아님)
└── 플레이어가 더 많은 정보로 더 나은 결정을 내릴 수 있음

보석 소비 — 가챠가 아닌 이유

보석으로 구매하는 모든 항목은 결과가 사전에 확정 표시된다. 확률 뽑기 요소가 전혀 없다.

소비 항목결과 예측 가능성판단
즉시 도착100% 확정 (배가 즉시 도착)확정
선원 고용등급·능력치 사전 표시 후 구매확정
선원 업그레이드업그레이드 결과 확정 표시확정
세금 면제100% 확정 (다음 세금 1회 면제)확정
이벤트 회피100% 확정 (이벤트 무효화)확정

첫 30분 플레이 설계

⏱️
비판: 게임 초반 유저 이탈 위험 — 너무 느린 진행, 불명확한 보상
설계 목표: 첫 30분 안에 "출발 → 방치 → 도착 → 거래" 루프를 1회 완료하고 자발적으로 다시 배를 출발시키게 만든다.

분 단위 시나리오

0~3분
세계관 진입
세계관 슬라이드 3장 표시 (스킵 가능)
→ 시작 자금 600금 + 보석 5개 지급
→ 화면: 런던 항구 줌인, 첫 퀘스트 없음 (깨끗한 시작)
★ 설계 포인트: 시작 자금 600금 — 슬루프(500금) + 선원 고용(50금) + 여유 50금. 첫 행동을 즉시 가능하게 한다.
3~7분
첫 번째 배 구매 tutorialStep 1
함선 탭 하이라이트: "슬루프를 구매하세요 (500금)"

7~10분
첫 선원 고용 tutorialStep 2

→ 일반 등급 항해사 1명 미리 배치 (50금 고용 유도)
→ 고용 완료: 선원 카드에 이름·효과 표시
10~13분
첫 항해 출발 tutorialStep 3

→ 추천 목적지 배지: "리스본 (15분)" — 가장 가까운 항구 강조
→ 출발 완료: 항로 골드 점선 애니메이션 시작
→ 팝업: "배가 출발했습니다! 앱을 꺼도 배는 항해합니다 ⚓"
13~28분
첫 방치 구간 의도된 공백
타이머 배지: "슬루프 1호 — 리스본까지 12분"
→ 플레이어는 탭 구경, 가격 차트 확인 자유롭게 탐색
→ 이 시간에 세금 타이머 미표시 (압박 없는 탐색 유도)
★ 설계 포인트: 방치를 "버그"가 아니라 "기능"임을 명확히 인지시키는 구간. 이 시간에 이탈하면 알림으로 복귀 유도.
28~30분
첫 도착 + 첫 거래 tutorialStep 4
로그 최상단: "★ 슬루프 1호가 리스본에 도착했습니다!"
→ 화물 탭 하이라이트: "자원을 구매해보세요"
→ 리스본에서 판매가 높은 자원 1종 강조 표시 (초보자 힌트)
→ 구매 완료 → 튜토리얼 종료 보상:
💰 500금 지급 💎 보석 2개 지급 첫 퀘스트 자동 부여
30분 이후
자유 플레이 루프 완성
플레이어는 이미 "출발 → 방치 → 도착 → 거래" 루프를 1회 완료
→ 첫 퀘스트: "긴급 배송 — 면화 10개를 런던으로" (쉬운 난이도)
→ 세금 첫 납부까지 약 20시간 남음 (충분한 여유)

모의 주식과 Pioneer의 구조적 유사성

모의 주식 앱이 높은 접속률을 갖는 이유는 단순히 수익 때문이 아니다. 핵심은 불확실성 → 결정 → 결과 확인이라는 심리적 사이클이다. Pioneer는 이 사이클을 무역 게임으로 정확히 재현한다.

모의 주식 앱Pioneer
시세 등락 차트를 실시간 확인항구별 자원 가격 히스토리 차트 확인

Pioneer만의 도파민 유발 설계 포인트

① 시세 변동 배지 (⚡)

앱을 닫은 사이 시세가 바뀌었다는 알림 → "얼마나 바뀌었지?" 즉각 접속 유도. 모의 주식 앱의 "삼성전자 +3.2%" 푸시 알림과 동일한 심리 자극.

② 배 도착 타이머

"슬루프 1호 — 리스본까지 12분" → 12분 후 결과를 확인하려는 재방문 유도. 쿠키런·클래시의 "완성까지 N분" 루프와 동일한 구조.

③ 세금 카운트다운

"다음 세금 납부까지 4h 30m" → 시간 압박이 주기적 접속을 강제. "오늘 세금 낼 돈 있나?" 확인 행동 자체가 습관 형성.

④ 퀘스트 기한 압박

"긴급 배송 — 2시간 남음" → 마감 임박 심리 (FOMO). 할 일을 미룰 수 없게 만드는 데드라인 도파민.

⑤ 세금 레벨 상승 경고

배 구매·판매 마일스톤 돌파 순간 "세금 레벨 상승!" 알림. 성장의 쾌감과 동시에 새로운 위기감 → 즉각적인 전략 재계산 유도.
