## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | v2.0 세금 시스템이 성장을 벌칙으로만 느끼게 함; 정보 ROI 음수; 이벤트가 방치형 게임과 충돌; 선원 역할 개념이 직관적이지 않음 |
| **Solution** | 세금 레벨 = 콘텐츠 해금 트리거; 정보를 가격 예측 전용으로 단순화; 이벤트를 접속 시 일괄 처리; 역할 제거 후 스탯 직접 표시; 로그라이트 레벨업으로 선단 성장 |
| **Core Loop 변화** | 배 출발 → 방치 → (이벤트 발생 시 +1h 대기) → 접속 → 이벤트 선택 → 도착 → 배달 정산 + 화물 판매 → 레벨업 → 업그레이드 선택 → 재출발 |

## 1. 세금 해금 시스템

### 1.1 설계 의도

세금 레벨 상승 = 성장의 트레이드오프. 압박과 동시에 새로운 콘텐츠가 열린다.
세금을 많이 낼수록 더 수익성 높은 품목과 더 강한 선원에 접근 가능해진다.

### 1.2 레벨별 해금 테이블

| 세금 레벨 | 해금 품목 | 해금 선원 풀 |
|---------|---------|-----------|
| L1~2 | 밀, 면화, 목재 (기초 3종) | 일반 등급만 |
| L3~4 | + 향신료, 비단, 도자기 | + 숙련 등급 등장 |
| L5~6 | + 상아, 염료, 금속 | + 전문가 등급 등장 |
| L7~8 | + 보석류, 희귀 향료 | + 전설 등급 등장 |
| L9+ | + 독점 품목 (세계 단일 항구만 취급) | 전설 풀 확장 |

### 1.3 UI 표시

- 아직 해금되지 않은 품목은 거래 패널에 잠금 아이콘으로 표시
- 잠금 품목 클릭 시 "세금 레벨 N 도달 시 해금" 툴팁 표시
- 레벨 상승 시 "새 품목 해금!" 로그 강조

## 2. 정보 시스템 개편

### 2.1 설계 의도

정보 구매 = 특정 항구의 특정 품목 가격 방향성 예측에 집중. 명확한 ROI로 전략적 구매를 유도한다.

### 2.2 예측 정보 형식

```
[정보 예시]
📈 리스본 · 향신료 — 2시간 내 +20~30% 예상  (정확도 65%)
📉 런던 · 비단 — 다음 시세 변동 시 -15% 예상  (정확도 45%)
```

### 2.3 등급별 상세

| 등급 | 비용 | 정확도 | 예측 내용 |
|------|------|--------|---------|
| 소문 | 무료 (매 시세 변동마다 3개 자동 지급) | 40% | 방향만 (상승/하락) |
| 귀띔 | 300금 | 60% | 방향 + 대략적 폭 |
| 분석 | 1,000금 | 75% | 방향 + 구체적 폭 + 시점 |
| 내부정보 | 4,000금 | 90% | 방향 + 구체적 폭 + 시점 + 지속 시간 |

### 2.4 ROI 설계

귀띔(300금) 적중 시 향신료 20개 기준 거래 차익 약 +800금 기대값 → 명확한 양수 ROI.
대량 거래 시 등급이 높을수록 기대 차익이 커짐.

### 2.5 이벤트 보상으로 정보 지급

이벤트 해결 후 자동으로 정보 1개 지급:
- 가벼운 이벤트 (풍랑 등) → 귀띔 1개
- 심각한 이벤트 (해적, 대공황 등) → 분석 1개

## 3. 이벤트 시스템 개편

### 3.1 설계 의도

방치형 게임과 충돌하는 실시간 팝업 방식을 제거. 이벤트는 접속 시 선택지로 처리하되, 항로당 1회로 제한하고 장거리 항로에 집중 발생시킨다.

### 3.2 발생 조건

- **항로당 최대 1회**
- **발생 확률** = `항해 시간(시간) × 8%`
  - 1시간 항로: 8%
  - 3시간 항로: 24%
  - 6시간 항로: 48%
  - 최대 80% 상한

### 3.3 이벤트 흐름

```
이벤트 발생 판정 (출발 시 1회)
  ↓
이벤트 발생 → 항해 시간 +1시간 자동 추가
  ↓
플레이어 접속 시 선택지 팝업 표시 (1시간 유효)
  ↓
[선택 시] 해당 결과 적용 + 정보 보상 지급
[미선택 시, 1시간 경과] 랜덤 선택 자동 처리
              → 접속 시 "자동 처리됨" 로그로 확인 가능
```

### 3.4 이벤트 선택지 테이블

| 이벤트 | 아이콘 | 선택지 A | 선택지 B | 선택지 C |
|--------|--------|---------|---------|---------|
| 풍랑 | ⛈️ | 우회 항로 (추가 +2시간) | 강행 (화물 10% 손실) | 💎 1개로 회피 |
| 해적 | 🏴‍☠️ | 협상 (금화 500 지불) | 저항 (화물 20% 손실) | 💎 2개로 회피 |
| 대공황 | 📉 | 목적지 변경 (가장 가까운 항구) | 감수 (도착 후 판매가 -30%) | 💎 3개로 회피 |
| 빙하 충돌 | 🧊 | 서행 항해 (추가 +3시간) | 강행 (선체 -50) | 💎 2개로 회피 |
| 역병 창궐 | ☠️ | 회항 (출발 항구 복귀) | 감수 (목적지 입항 1회 불가) | 💎 2개로 회피 |
| 순풍 행운 | 💨 | 항해 시간 -30% (자동 적용, 선택 불필요) | — | — |

### 3.5 이벤트 보상

모든 이벤트 해결 후 자동 정보 지급:
- 풍랑·빙하·역병 → 귀띔 1개
- 해적·대공황 → 분석 1개
- 순풍 → 보상 없음 (이미 시간 단축)

## 4. 로그라이트 레벨 시스템

### 4.1 설계 의도

매매·세금 납부가 경험치로 연결되어 레벨업 시 선단 전체를 성장시키는 영구 업그레이드를 선택한다. Slay the Spire 스타일 — 3개 카드 중 1개 선택, 영구 적용. 시즌 리셋은 개발자가 설정.

### 4.2 경험치 획득

| 행동 | 경험치 |
|------|--------|
| 화물 판매 | 판매금액 × 0.1 XP |
| 화물 구매 | 구매금액 × 0.05 XP |
| 세금 납부 | 납부금액 × 0.2 XP |
| 퀘스트 완료 | 200 XP 고정 |
| 일일목표 완료 | 50 XP 고정 |
| 배달 의뢰 완료 | 30 XP 고정 |

**레벨업 임계값**: `레벨 × 500 XP`
- L1→L2: 500 XP
- L2→L3: 1,000 XP
- L10→L11: 5,000 XP

### 4.3 업그레이드 카드 풀

레벨업 시 아래 풀에서 랜덤 3장을 뽑아 1개 선택. 중복 선택 가능 (수치 누적).

| 카테고리 | 업그레이드 예시 |
|---------|--------------|
| 함대 | 모든 배 항속 +5% / 화물칸 +3개 / 연료 소모 -8% |
| 경제 | 거래 수수료 -2% / 구매가 -3% / 판매가 +3% |
| 생존 | 이벤트 발생률 -3% / 세금 납부 유예 +2h / 선체 수리비 -10% |
| 선원 | 선원 슬롯 +1 / 선원 고용 비용 -10% / 전체 선원 스탯 +3% |
| 정보 | 무료 소문 +1개/시세변동 / 정보 정확도 +5% / 정보 구매 비용 -15% |

### 4.4 레벨업 UI 흐름

```
레벨업 달성
  ↓
"레벨 N 달성!" 헤더 배지 표시
  ↓
업그레이드 카드 3장 팝업 (화면 중앙)
  ↓
플레이어 선택 → 즉시 적용 + 로그 기록
```

### 4.5 시즌 리셋

- 시즌 종료 시: 레벨·업그레이드 초기화
- 시즌 보상: 누적 레벨에 따라 보석·선원 등 지급
- 리셋 시기는 개발자가 서버 또는 상수값으로 설정

## 5. 퀘스트 실패 패널티 변경

### 5.1 설계 의도

실패 시 즉각 타격(세금 1회분 추가) 대신, 서서히 목이 조이는 영구 세금 증가. 무작정 수락보다 신중한 선택을 유도.

### 5.2 패널티 공식

```
실패 패널티: 현재 일일 세금 × +2% 영구 추가
누적 상한: +20% (10회 실패까지만 누적)

예)
세금 1,000금 → 실패 1회 → 1,020금
             → 실패 2회 → 1,040금
             → 실패 10회 → 1,200금 (상한)
```

### 5.3 패널티 완화

- 특수 퀘스트 보상으로 "세금 패널티 -1회분" 지급 가능 (희귀)
- 세금 레벨이 오를 때 패널티 누적치 초기화 (레벨업 = 새 구간)

## 6. 퀘스트 / 일일목표 분리

### 6.1 분류 기준

| 구분 | 퀘스트 | 일일목표 |
|------|--------|---------|
| 성격 | 스토리형, 도전형 | 습관형, 루틴 |
| 기한 | 2~4시간 (엄격) | 자정까지 (24h) |
| 보상 | 금화(세금 2~3배) + 정보 1개 | 소액 금화 + 경험치 |
| 실패 패널티 | 세금 영구 +2% | 없음 |
| 수락 여부 | 플레이어 선택 (거절 가능) | 자동 부여 |
| 최대 동시 | 3개 | 4개 |

### 6.2 일일목표 예시

| 목표 | 보상 |
|------|------|
| 배 1척 이상 목적지 출발 | 100금 + 20 XP |
| 화물 10개 이상 판매 | 200금 + 30 XP |
| 이벤트 선택지 처리 | 150금 + 25 XP |
| 선원 1명 고용 | 100금 + 20 XP |

### 6.3 UI 분리

- 하단 패널에 [퀘스트]와 [목표] 탭을 별도로 분리
- 퀘스트: 기한 카운트다운, 붉은 경고 표시
- 일일목표: 진행 바, 자정 리셋 타이머

## 7. 선원 스탯 시스템

### 7.1 설계 의도

역할(항해사/상인 등) 레이블 제거. 각 선원이 보유한 스탯 수치를 직접 표시하여 플레이어가 숫자만 보고 즉시 판단 가능하게 한다.

### 7.2 스탯 5종

| 스탯 | 아이콘 | 효과 |
|------|--------|------|
| 항속 | ⚡ | 배 이동 속도 +N% |
| 적재 | 📦 | 화물 칸 +N개 |
| 거래 | 💰 | 구매가 -N/2%, 판매가 +N/2% |
| 방어 | 🛡️ | 이벤트 발생 확률 -N% |
| 정비 | 🔧 | 수리비 -N/2%, 연료 소모 -N/2% |

### 7.3 등급별 스탯 구성

| 등급 | 보유 스탯 수 | 수치 범위 |
|------|------------|---------|
| 일반 | 1개 | +5~10% (또는 +3~5칸) |
| 숙련 | 2개 | +10~18% |
| 전문가 | 3개 | +18~28% |
| 전설 | 3~4개 | +28~40% |

### 7.4 선원 카드 예시

```
[이반 코르테스]  🟪 전문가  출신: 리스본
⚡ 항속  +22%
💰 거래  +14%
🛡️ 방어  +6%     ← 리스본 출신 보너스
```

### 7.5 중첩 공식 (수확 체감 유지)

```js
// 같은 스탯 N명 중첩
const stackedEffect = 1 - Math.pow(1 - individualEffect, crewCount);
```

## 8. 항구 선원 고용 시스템

### 8.1 출신 항구 보너스

각 항구는 고유한 보너스 스탯을 선원에게 부여. 그 항구에서 고용한 선원만 영구히 보유.
동일 출신 항구 선원의 보너스는 중첩되지 않아, 다양한 항구 방문이 최적 선원 구성에 필요.

| 항구 (예시) | 출신 보너스 스탯 |
|-----------|--------------|
| 런던 | 거래 +5% |
| 리스본 | 방어 +6% |
| 알렉산드리아 | 이벤트 해결 시 정보 등급 +1 (귀띔→분석) |
| 베이징 | 적재 +4칸 |
| 캘리컷 | 항속 +8% |
| 카이로 | 정비 +6% |
| 바스코 다가마 루트 항구 | 항속 +5% + 이벤트 발생률 -3% |

(항구별 보너스는 전체 항구 수에 맞게 설계 시 확장)

### 8.2 새로고침 시스템

항구 정박 중 고용 가능 선원 목록을 갱신. 비용은 매번 증가하며, 항구를 떠났다가 돌아오면 초기화.

| 새로고침 횟수 | 기본 비용 | 프리미엄 비용 |
|------------|---------|------------|
| 1회 | 200금 | 💎 1개 |
| 2회 | 400금 | 💎 1개 |
| 3회 | 800금 | 💎 1개 |
| 4회 | 1,600금 | 💎 2개 |
| 5회+ | 3,200금 (상한) | 💎 2개 |

- **기본 새로고침**: 일반/숙련 풀에서만 갱신
- **프리미엄 새로고침**: 전문가+ 풀 포함
- **초기화 조건**: 항구를 출항하면 횟수 카운터 리셋

## 9. 세금 미납 처리 개편

### 9.1 설계 의도

금화 부족 시 보석으로 즉시 대납하는 대신, 먼저 배를 강제 매각하여 자산으로 세금을 충당. 배가 1척뿐이면 매각 중단.

### 9.2 처리 순서

```
세금 납부 시도
  ↓
금화 부족?
  ↓
[1단계] 가장 저렴한 배부터 구매가의 50%에 강제 매각
        (납부액 충족될 때까지 반복)
  ↓
배 1척만 남은 경우 → 매각 중단
  ↓
[2단계] 여전히 부족 → 보석 자동 대납 (보석 1개 = 세금 1회분)
  ↓
[3단계] 보석도 없음 → 미납 누적 로그 (다음 납부 시 2배 징수)
```

### 9.3 매각 알림 UI

```
⚠️ 세금 납부 부족 — [슬루프 2호] 강제 매각
   구매가 3,000금 → 매각가 1,500금 수령
   잔여 세금: 0금 (완납)
```

### 9.4 사전 경고

- 세금 납부 4시간 전부터 헤더 세금 표시 빨강 전환
- 1시간 전: "배 매각 위험" 경고 배지 표시
- 플레이어가 미리 배를 수동 매각(50% 가격)하는 것도 가능

## 10. 항구 배달 의뢰 시스템

### 10.1 설계 의도

직접 매매가 아닌 화물칸을 빌려주는 방식. 빈 배로 이동해야 할 때 손실을 최소화하고, 다양한 항구 방문을 자연스럽게 유도하는 보조 수익원.

### 10.2 구조

```
항구 도착 → [배달 의뢰] 탭에서 의뢰 목록 확인
  ↓
의뢰 선택 (목적지·화물 종류·칸 수·보상 표시)
  ↓
출발 → 항해 중 의뢰 화물이 해당 칸 점유
  ↓
목적지 도착 시 자동 정산 + 경험치 지급
```

### 10.3 의뢰 예시

```
[런던 → 리스본]  면화 15칸  보상: 320금  ⏱ 2h
[런던 → 카이로]  목재 20칸  보상: 580금  ⏱ 6h
[런던 → 베이징]  향신료 8칸  보상: 1,200금  ⏱ 10h  ★ 고수익
```

### 10.4 상세 규칙

| 항목 | 내용 |
|------|------|
| 보상 | 동일 루트 직접 무역 대비 약 25~35% |
| 화물칸 | 의뢰 화물이 지정된 칸 수 점유, 나머지 칸은 자유 무역 가능 |
| 경험치 | 정산 시 30 XP 고정 |
| 실패 조건 | 목적지 변경 시 의뢰 자동 취소 (보상 없음, 패널티 없음) |
| 의뢰 갱신 | 항구 도착마다 새 의뢰 목록 생성 (0~3개) |
| 복수 의뢰 | 칸 수가 허용하면 같은 목적지 의뢰 동시 수락 가능 |

## 11. 시스템 간 상호작용 정리

| 시스템 A | 시스템 B | 상호작용 |
|---------|---------|---------|
| 세금 레벨 | 해금 시스템 | 레벨 상승 시 품목·선원 풀 자동 해금 |
| 이벤트 | 정보 시스템 | 이벤트 해결 시 정보 자동 지급 |
| 이벤트 | 선원 방어 스탯 | 방어 스탯 합계만큼 이벤트 발생률 감소 |
| 퀘스트 실패 | 세금 | 실패 누적 시 기본 세금 영구 증가 |
| 로그라이트 레벨 | 선원 고용 | "선원 슬롯 +1" 업그레이드 선택 시 배 전체에 적용 |
| 배달 의뢰 | 이벤트 | 이벤트 발생 시 배달 화물도 선택지 영향 받음 (우회 → 배달 취소 없음, 강행 → 화물 손실에 배달분 포함) |
| 항구 출신 보너스 | 로그라이트 | "전체 선원 스탯 +N%" 업그레이드는 출신 보너스 포함 전체에 적용 |

## 12. 데이터 구조 변경사항

### 12.1 CrewMember 변경

```js
// v2.0
{
  role: 'navigator' | 'merchant' | 'cargo' | 'engineer',
  primaryStat: number,
}

// v2.1
{
  stats: {
    speed:   number,  // % (0이면 해당 스탯 없음)
    cargo:   number,  // 칸 수
    trade:   number,  // %
    defense: number,  // %
    repair:  number,  // %
  },
  originPort: string,       // 출신 항구 키
  originBonus: {            // 출신 항구 보너스 (고용 시 부여, 영구)
    statKey: string,
    value: number,
  },
}
```

### 12.2 GameState 추가 필드

```js
{
  // 로그라이트
  playerLevel: number,
  playerXP: number,
  permanentUpgrades: string[],  // 선택된 업그레이드 ID 배열

  // 퀘스트 실패 패널티
  taxPenaltyPct: number,        // 0~20, 기본 세금에 추가되는 %

  // 이벤트 큐
  pendingEvents: PendingEvent[], // 접속 전 처리 대기 중인 이벤트
}

// PendingEvent
{
  shipId: string,
  eventType: string,
  choices: EventChoice[],
  expiresAt: number,   // timestamp (발생 후 1시간)
  autoResolved: boolean,
  autoChoice: string | null,
}
```

### 12.3 Port 데이터 추가

  },

## 13. 성공 기준

| 기준 | 목표 |
|------|------|
| 세금 해금 | 세금 L3 도달 시 향신료·비단 거래 패널에 표시, L5에서 전문가 선원 등장 |
| 정보 ROI | 귀띔 1개 적중 시 평균 수익이 비용 300금 초과 |
| 이벤트 처리 | 오프라인 중 이벤트 발생 후 접속 시 선택지 팝업 정상 표시 |
| 이벤트 자동 처리 | 1시간 무선택 시 랜덤 처리 + 로그 기록 정상 동작 |
| 로그라이트 | 레벨업 시 카드 3장 정상 표시, 선택 후 즉시 효과 적용 |
| 퀘스트 패널티 | 실패 시 taxPenaltyPct 정확히 +2 증가, 상한 20 준수 |
| 선원 스탯 | 역할 레이블 없이 스탯 수치만 표시, 배정 시 정확히 선단에 반영 |
| 항구 보너스 | 출신 항구 보너스가 선원에 영구 귀속, 이적 후에도 유지 |
| 새로고침 비용 | 횟수마다 비용 정확히 증가, 출항 시 초기화 |
| 세금 미납 | 배 강제 매각 → 보석 → 미납 순서 정확히 실행 |
| 배달 의뢰 | 목적지 도착 시 자동 정산, 화물칸 점유 정확히 처리 |
