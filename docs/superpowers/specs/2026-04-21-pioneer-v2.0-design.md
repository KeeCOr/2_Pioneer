# Pioneer Game v2.0 — Design Document

**Feature**: pioneer-game-v2.0
**Phase**: Design
**Date**: 2026-04-21
**Status**: Draft
**Base Version**: pioneer-game (v1.0 97% Match) + 현재 구현 (1417줄, v2.x 기능 포함)
**Version Strategy**: Option B — 현재 구현을 v2.0 공식 베이스로 인정, 신규 시스템 추가

---

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 엔드게임 없음, 세션 설계 없음, 수익화 미구현, FTUE 부재, 선원 역할 단조로움 |
| **Solution** | 생존형 엔드게임(지수 세금) + 방치형 루프 + 보석 5종 수익화 + 이벤트/퀘스트 시스템 + 선원 역할 분화 + 인터랙티브 튜토리얼 |
| **Core Loop** | 배 출발 → 방치 → 도착 알림 → 접속 → 거래·퀘스트·이벤트 처리 → 다음 항해 출발 |
| **Core Value** | "세금에 맞서 얼마나 오래 살아남는가" — 생존이 목표, 방치가 플레이 방식 |

## 2. 핵심 게임 루프

### 2.1 방치형 루프

```
[접속]
  ↓
도착한 배 확인 → 화물 판매
  ↓
퀘스트 처리 (완료/실패 판정)
  ↓
이벤트 확인 (팝업 처리)
  ↓
다음 항구로 배 출발
  ↓
[방치 — 앱 종료 가능]
  ↓
1시간마다 시세 변동 (헤더 배지 "⚡ 시세 변동됨")
배 도착 시 로그 기록
  ↓
[재접속]
```

**방치 설계 원칙**
- 항해 중 플레이어 행동 없음 (의도적 공백)
- 도착 알림: 로그 최상단 강조 표시
- 시세 변동 배지: 헤더에 ⚡ 아이콘 표시, 클릭 시 배지 제거

### 2.2 세션 길이별 경험

| 세션 | 행동 |
|------|------|
| 1~2분 | 도착 배 화물 판매 + 다음 항구 출발 |
| 5~10분 | 위 + 퀘스트 확인 + 선원 관리 |
| 20~30분 | 위 + 정보 구매 + 배 업그레이드 + 이벤트 대응 |

## 3. 생존 시스템 (세금)

### 3.1 세금 공식

```js
const TAX_INTERVAL = 24 * 60 * 60; // 86400초 = 실제 1일

// 누적 판매금 마일스톤 (돌파 시 taxLevel +1)
const EARN_MILESTONES = [10000, 50000, 200000, 800000, 3000000];

// 레벨별 일일 세금 (구간 내 고정, 구간 돌파 시 급등)
const TAX_TABLE = [200, 600, 1000, 3000, 7000, 20000, 50000, 120000, 300000, 750000];
const calcTax = (_shipCount, taxLevel) =>
  TAX_TABLE[Math.min(taxLevel - 1, TAX_TABLE.length - 1)];
```

**설계 의도**: 구간 내에서는 거의 느껴지지 않다가, 배를 사거나 거래가 성장하는 순간 세금이 껑충 뛴다. 성장의 대가를 플레이어가 명확히 느끼게 한다.

### 3.2 세금 레벨 기대값

| 레벨 | 일일 납부액 | 진입 조건 | 느낌 |
|------|-----------|----------|------|
| 1 | 200금 | 게임 시작 | 거의 무료 |
| 2 | 600금 | 배 2번째 구매 | 살짝 쓴맛 |
| 3 | 1,000금 | 누적 1만금 판매 | 슬슬 신경 쓰임 |
| 4 | 3,000금 | 배 3번째 구매 | 압박 시작 |
| 5 | 7,000금 | 누적 5만금 판매 | 본격 경영 |
| 6 | 20,000금 | 배 4번째 구매 | 유의미한 부담 |
| 7 | 50,000금 | 누적 20만금 판매 | 긴장감 |
| 8 | 120,000금 | 배 5번째 구매 | 극한 압박 |
| 9 | 300,000금 | 누적 80만금 판매 | 생존 경영 |
| 10 | 750,000금 | 누적 300만금 판매 | 전설의 상인 |

### 3.3 세금 레벨 상승 조건

- **배 추가 구매 시** → +1레벨 (구매 즉시)
- **누적 판매금 마일스톤 돌파 시** → +1레벨 (1만 / 5만 / 20만 / 80만 / 300만금)
- **납부 자체로는 레벨 상승 없음** — 성장하지 않으면 세금도 오르지 않는다

### 3.4 납부 주기 및 미납 처리

- **납부 주기**: 실제 24시간마다 1회 자동 징수 (마지막 납부 기준, 자정 아님)
- **저장 연동**: `lastTaxAt` 타임스탬프를 localStorage에 저장 → 새로고침 후에도 남은 시간 유지
- **미납 시**: 금화 부족 → 보석 1개 자동 대납, 보석도 없으면 로그만 출력 (압류는 v2.1)
- **헤더 표시**: "Lv.N — 일일 Xh YYm" 형식으로 다음 징수까지 남은 시간 표시

### 3.5 세금 면제 (보석)

- 보석 2개 소비 → 다음 세금 1회 면제
- 헤더에 면제 상태 표시: 🛡️ 세금 면제 중

## 4. 이벤트 시스템

### 4.1 발생 구조

- **판정 시점**: 시세 갱신(1시간)마다 항해 중인 배 각각에 독립 판정
- **기본 발생 확률**: 배당 15%
- **항해사 선원 효과**: 배당 이벤트 확률 -5% (항해사 1명당)

### 4.2 이벤트 테이블

| 이벤트 | 아이콘 | 가중치 | 효과 | 회피 비용 |
|--------|--------|--------|------|-----------|
| 풍랑 | ⛈️ | 30 | 도착 시간 +50%, 각 자원 수량 5~15% 랜덤 손실 | 💎 1개 |
| 해적 출몰 | 🏴‍☠️ | 25 | 화물 20~40% 강탈 | 💎 2개 |
| 대공황 | 📉 | 15 | 다음 시세 전 항구 -30% | 💎 3개 |
| 빙하 충돌 | 🧊 | 15 | hull -50, 속도 -30% (수리 전까지) | 💎 2개 |
| 역병 창궐 | ☠️ | 10 | 목적지 항구 입항 불가 (1회 갱신) | 💎 2개 |
| 순풍 행운 | 💨 | 5 | 도착 시간 -30% (긍정, 회피 불필요) | — |

### 4.3 이벤트 UI 흐름

```
이벤트 발생
  ↓
EventOverlay 팝업 (화면 중앙):
"⛈️ [슬루프 1호]가 풍랑을 만났습니다!"
[💎 1개로 회피]  [감수하기]
  ↓
감수 → 효과 즉시 적용 + 로그 기록
회피 → 보석 차감, 이벤트 무효 + 로그 기록
```

### 4.4 선체 내구도 시스템

```js
// Ship 필드 추가
hull: number  // 0~100, 초기 100

// 빙하 충돌 시
hull -= 50;
if (hull <= 0) { /* 배 침몰 — 화물 전량 손실, ships에서 제거 */ }

// 수리 비용 (항구 도착 시)
const repairCost = (100 - ship.hull) * 50;
```

## 5. 퀘스트 시스템

### 5.1 퀘스트 생성

- **생성 시점**: 항구 도착 시 0~2개 랜덤 생성
- **최대 동시 진행**: 3개
- **수락 여부**: 플레이어 선택 (거절 시 소멸, 패널티 없음)

### 5.2 퀘스트 종류

| 퀘스트 | 조건 | 제한 | 성공 보상 | 실패 패널티 |
|--------|------|------|-----------|-------------|
| 긴급 배송 | 자원 10~30개 → 지정 항구 | 2시간 | 금화(세금 2배) + 💎1 | 세금 1회분 추가 |
| 항구 방문 | 3~5개 항구 탐방 | 4시간 | 금화(세금 3배) + 특수 선원 1명 | 세금 1회분 추가 |
| 무역왕 | 특정 자원으로 5,000~20,000금 수익 | 3시간 | 금화(세금 2배) + 세금 레벨 -1 | 세금 1회분 추가 |
| 위기 구조 | 폭풍 이벤트 중 배 1척 귀환 | 1시간 | 💎 2개 | 세금 1회분 추가 |
| 독점 계약 | 특정 자원 20~50개 매입 | 2시간 | 금화(세금 2배) + 해당 자원 판매가 +20% (1회) | 세금 1회분 추가 |

### 5.3 보상 설계 원칙

- 금화 보상: 현재 세금 1회분의 2~3배 (항상 수락 가치 있음)
- 보석 보상: 어려운 퀘스트에만 포함 (주요 무과금 획득 경로)
- 세금 레벨 -1: 희귀 보상, 최강 동기 부여

### 5.4 퀘스트 데이터 구조

```js
{
  id: string,
  type: 'delivery' | 'visit' | 'trade' | 'rescue' | 'monopoly',
  description: string,
  condition: {},        // 퀘스트별 달성 조건
  deadlineAt: number,  // timestamp
  reward: { gold, gems, crewId, taxLevelDelta },
  penalty: { taxMultiplier: 1 },  // 세금 1회분 추가
  status: 'active' | 'completed' | 'failed',
}
```

## 6. 선원 특화 시스템

### 6.1 역할 4종

| 역할 | 아이콘 | 주 효과 | 부 효과 |
|------|--------|---------|---------|
| 항해사 | ⚓ | 배 속도 +N% | 이벤트 발생률 -5% |
| 상인 | 💰 | 구매가 -N%, 판매가 +N% | — |
| 화물관리사 | 📦 | 수납량 +N칸 | 연료 소모 -5% |
| 기관사 | ⚙️ | 연료 효율 +N% | 선체 수리비 -10% |

### 6.2 등급 4단계

| 등급 | 색상 | 주 효과 범위 | 고용 방법 |
|------|------|-------------|-----------|
| 일반 | ⬜ | +5~10% | 금화 (항구) |
| 숙련 | 🟦 | +10~18% | 금화 (항구, 드묾) |
| 전문가 | 🟪 | +18~28% | 💎 1~2개 |
| 전설 | 🟧 | +28~40% | 💎 3개 + 특수 퀘스트 |

### 6.3 선원 업그레이드 (보석)

```
일반 → 숙련:    💎 1개
숙련 → 전문가:  💎 2개
전문가 → 전설:  💎 3개
```

### 6.4 중첩 공식 (수확 체감)

```js
// 같은 역할 선원 N명 효과 중첩
const stackedEffect = 1 - Math.pow(1 - individualEffect, crewCount);
```

### 6.5 선원 데이터 구조

```js
{
  id: string,
  name: string,
  role: 'navigator' | 'merchant' | 'cargo' | 'engineer',
  grade: 'common' | 'skilled' | 'expert' | 'legendary',
  primaryStat: number,    // 주 효과 수치 (소수, 예: 0.12 = 12%)
  shipId: string | null,
  hireCost: number,
}
```

## 9. 저장/불러오기 시스템

```js
// 저장 키
const SAVE_KEY = 'pioneer_save_v2';

// saveGame()
const saveData = {
  saveVersion: '2.0',
  savedAt: new Date().toISOString(),
  gs,
  priceHistory,
};
localStorage.setItem(SAVE_KEY, JSON.stringify(saveData));

// loadGame() — 초기화 시 1회
const raw = localStorage.getItem(SAVE_KEY);
if (!raw) return null;  // 새 게임
const data = JSON.parse(raw);
if (data.saveVersion !== '2.0') return null;  // 구버전 무시
setGs(data.gs);
setPriceHistory(data.priceHistory ?? {});

// 자동 저장: 시세 갱신마다 + 항구 도착마다
```

## 13. 도파민 설계 — 모의 주식급 접속률 목표

> **Pioneer는 모의 주식 앱과 동일한 수준의 도파민과 재방문율을 목표로 설계된다.**

### 13.1 모의 주식과 Pioneer의 구조적 유사성

모의 주식 앱이 높은 접속률을 갖는 이유는 단순히 "돈을 벌기 때문"이 아니다. 핵심은 **불확실성 → 결정 → 결과 확인**이라는 심리적 사이클이다. Pioneer는 이 사이클을 무역 게임으로 정확히 재현한다.

| 모의 주식 앱 | Pioneer |
|------------|---------|
| 시세 등락 차트를 실시간 확인 | 항구별 자원 가격 히스토리 차트 확인 |
| "이 종목 지금 사야 해?" 결정 긴장감 | "리스본에서 비단 지금 사야 해?" 결정 긴장감 |
| 매수 후 다음 시세 확인하러 재접속 | 배 출발 후 도착 결과 확인하러 재접속 |
| 포트폴리오 수익률 확인 | 함대 운영 수익 확인 |
| 손실이 두려워 눌러보게 됨 | 세금 납부 가능한지 확인하러 눌러보게 됨 |

### 13.2 Pioneer만의 도파민 유발 설계 포인트

**① 시세 변동 배지 (⚡)**
- 앱을 닫은 사이 시세가 바뀌었다는 알림 → "얼마나 바뀌었지?" 즉각 접속 유도
- 모의 주식 앱의 푸시 알림 "삼성전자 +3.2%" 와 동일한 심리 자극

**② 배 도착 타이머**
- "슬루프 1호 — 리스본까지 12분" → 12분 후 결과를 확인하려는 재방문 유도
- 쿠키런·클래시의 "완성까지 N분" 루프와 동일한 구조

**③ 세금 카운트다운**
- "다음 세금 납부까지 4h 30m" → 시간 압박이 주기적 접속을 강제
- "오늘 세금 낼 돈 있나?" 확인 행동 자체가 습관 형성

**④ 퀘스트 기한 압박**
- "긴급 배송 — 2시간 남음" → 마감 임박 심리 (FOMO)
- 할 일을 미룰 수 없게 만드는 데드라인 도파민

**⑤ 세금 레벨 상승 경고**
- 배를 구매하거나 판매 마일스톤을 돌파하는 순간 "세금 레벨 상승!" 알림
- 성장의 쾌감과 동시에 새로운 위기감 → 즉각적인 전략 재계산 유도

### 13.3 모의 주식 대비 Pioneer의 우위

| 항목 | 모의 주식 앱 | Pioneer |
|------|------------|---------|
| 실패 시 감정 | 허탈 (가상 손실도 심리적 타격) | 세금 위기 → 극복 서사로 전환 |
| 진입장벽 | 경제 지식 필요 | 튜토리얼 30분으로 완전 숙달 |

## 14. IP 독립성 선언

> **Pioneer는 런닝맨(Running Man)과 IP 상의 관련이 없습니다.**

Pioneer의 일부 게임 규칙 및 미션 구조는 런닝맨의 유명 게임 세그먼트에서 영감을 받았습니다. 그러나 이는 순수하게 **규칙 구조(mechanics)** 만을 참고한 것이며, 다음 사항은 Pioneer와 전혀 무관합니다.

| 항목 | 상태 |
|------|------|
| 런닝맨 상표·로고·브랜드 | ❌ 미사용 |
| 런닝맨 캐릭터·출연진 | ❌ 미사용 |
| 런닝맨 방송 콘텐츠·영상 | ❌ 미사용 |
| 런닝맨 명칭·타이틀 | ❌ 미사용 |

Pioneer는 독자적인 세계관(대항해시대 무역 생존), 독자적인 캐릭터, 독자적인 아트워크를 가진 **완전히 독립된 오리지널 게임**입니다.

게임 규칙 아이디어를 참고하는 것은 법적으로 보호되지 않으므로(아이디어 자체는 저작권 보호 대상 아님, 표현물만 보호) 이는 일반적인 게임 디자인 관행에 해당합니다. 체스 규칙을 참고해 새로운 보드게임을 만들거나, 포커 규칙을 참고해 카드 게임을 만드는 것과 동일한 원칙입니다.

| 콘텐츠 분량 | 표준 | 추가 항구 5개, 배 종류 2개 추가 |
| 업적 시스템 | 없음 | Steam 업적 30개 |
| 모딩 지원 | 없음 | constants/ 파일 공개 (커뮤니티 확장) |
| 저장 방식 | localStorage | Steam Cloud Save |

**스팀 판매 타겟 포지션**
- 태그: Idle, Strategy, Simulation, Trading, Management
- 비교 게임: Slay the Spire (간단하지만 깊은 전략), Port Royale (같은 소재)
- 출시 가격: ₩12,000 (얼리 액세스) → ₩18,000 (정식 출시)

## 14. 선원 수집 시스템 — 장기 플레이 설계

### 14.1 설계 철학

> **"선원 하나가 게임을 바꾼다"** — 선원은 단순 보조 수치가 아니라 플레이어가 능동적으로 수집·조합·육성하는 핵심 콘텐츠다.

- 선원 총 풀: 약 **50~80명** (이름·특기·능력치 전부 고유)
- 같은 역할이라도 세부 특기가 달라 수집 동기 부여
- 전설 등급 선원은 특수 조건(퀘스트 완료, 특정 항구 방문)으로만 등장

### 14.2 확장 능력치 구조

| 능력치 | 기본 범위 | 특기 조합 시 |
|--------|---------|-----------|
| 속도 보정 | +5~40% | 항해사 + 기관사 조합 시 추가 시너지 +5% |
| 화물 보정 | +5~30칸 | 화물관리사 2인 조합 시 +5칸 추가 |
| 판매가 보정 | +5~35% | 항구 친밀도가 높은 상인 선원: +10% 추가 |
| 이벤트 회피 | -5~25% | 항해사 전문가 이상: 폭풍 회피 -10% 추가 |
| 연료 효율 | +5~30% | 기관사 2인 조합: 연료 소모 -15% 추가 |
| 수리비 할인 | -10~40% | 기관사 전설: 항구 수리 무료 1회/일 |

### 14.3 선원 시너지 조합 예시

| 조합 | 보너스 효과 | 획득 난이도 |
|------|-----------|-----------|
| 항해사 + 기관사 (같은 선박) | 속도 +8%, 연료 -10% | 보통 |
| 상인 3인 편대 | 판매가 +5% 추가 중첩 | 어려움 |
| 전설 선원 2인 | 특수 항로 해금 (고수익 지역) | 매우 어려움 |
| 지역 전문 선원 (아라비아 상인 등) | 해당 지역 시세 미리 보기 | 특수 퀘스트 |

### 14.4 선원 획득 루트

```
일반 등급: 항구 도착 시 고용 가능 (금화)
숙련 등급: 드물게 등장 (금화), 일부 퀘스트 보상
전문가 등급: 보석 1~2개 or 특수 퀘스트 완료
전설 등급: 특정 항구 누적 방문 10회 이상 or 퀘스트 체인 완료
지역 전문가: 해당 지역 교역량 임계값 돌파 시 해금
```

### 14.5 배 구매 비용 재설계 (장기 플레이 유도)

> 배 구매는 즉시 결정이 아니라 **며칠간 저축하는 목표**가 되어야 한다.

| 배 종류 | 재설계 비용 | 예상 저축 기간 | 세금 증가 |
|--------|-----------|------------|---------|
| 슬루프 | 500금 (튜토리얼) / 5,000금 (이후) | ~5일 | L+1 |
| 상인선 | 35,000금 | ~2주 | L+1 |
| 갤리온 | 80,000금 | ~3주 | L+1 |
| 프리깃 | 150,000금 | ~4~5주 | L+1 |
| 전함 (최고급) | 300,000금 | ~2개월 | L+1 |

**설계 의도**: 배 구매 직전 2~3일이 게임에서 가장 긴장감 있는 구간이다. "이번 세금 납부하고도 충분한가?"를 매일 계산하게 만든다.

## 14. 도박성 분류 회피 — 설계 근거

### 14.1 핵심 구분: 경제 불확실성 vs 확률 베팅

앱스토어 가이드라인(Apple 4.3, Google Play 도박 정책)은 **"결과가 무작위이며 실제 화폐로 베팅하는 행위"**를 도박으로 정의한다. Pioneer의 가격 변동은 다음 이유로 이 정의에 해당하지 않는다.

| 구분 | 도박 게임 | Pioneer |
|------|-----------|---------|
| 결과 결정 방식 | 순수 난수 (RNG) | 수급 알고리즘 (결정론적) |
| 플레이어 영향력 | 없음 | 루트 선택·타이밍·선원 구성으로 수익 차이 발생 |
| 실패 반복 가능성 | 동일 확률 반복 | 정보 구매·경험 축적으로 개선 가능 |
| 실제 화폐 베팅 | 있음 | 없음 (인앱 결제는 보석 구매이며 게임 결과에 직접 베팅 아님) |

### 14.2 가격 변동 설계 — 투명성 원칙

```
수급 알고리즘 공개 원칙
├── 각 자원의 현재 가격은 언제나 화면에 표시
├── 가격 히스토리(차트)를 통해 추세 파악 가능
├── "정보 구매" 시스템은 불확실성을 줄이는 수단 (완전 제거 아님)
└── 플레이어가 더 많은 정보로 더 나은 결정을 내릴 수 있음
```

이는 Stardew Valley 작물 시세, Civilization 교역 시스템과 동일한 **경제 시뮬레이션** 범주다.

### 14.3 보석 소비 — 가챠 아님

| 보석 소비 항목 | 결과 예측 가능성 |
|---------------|----------------|
| 즉시 도착 | 100% 확정 (배가 즉시 도착) |
| 선원 고용 | 등급·능력치 사전 표시 후 구매 |
| 선원 업그레이드 | 업그레이드 결과 확정 표시 |
| 세금 면제 | 100% 확정 (다음 세금 1회 면제) |
| 이벤트 회피 | 100% 확정 (이벤트 무효화) |

**랜덤 박스, 확률 뽑기, 숨겨진 확률표 없음.** 모든 보석 소비는 결과가 사전에 명시된다.

### 14.4 앱스토어 분류 전략

- 장르 태그: **Strategy / Simulation** (Gambling/Casino 카테고리 아님)
- 연령 등급: 4+ (실제 화폐 베팅 없음 → 도박 관련 등급 상승 사유 없음)
- 스크린샷 방향: 지도·무역 화면 중심, 가격 차트는 "경제 전략" 맥락으로 설명

## 15. 경쟁 게임 포지셔닝

### 15.1 시장 내 위치

```
                     높은 전략성
                          │
            Port Royale   │   Pioneer ◄ 우리
            (복잡, PC)    │   (단순·깊이 균형, 모바일)
                          │
낮은 방치 ─────────────────┼───────────────── 높은 방치
                          │
         Clash of Clans   │   Merge Mansion
         (전쟁·건설)       │   (수집·퍼즐)
                          │
                     낮은 전략성
```

### 15.2 주요 경쟁작과의 차이

| 게임 | 핵심 루프 | Pioneer와의 차이 |
|------|-----------|----------------|
| Shop Titans | 아이템 제작 → 영웅에게 판매 → 장비 강화 | 생산 없음, **루트 최적화와 시세 판단**이 핵심 |
| Merge Mansion | 오브젝트 합성 → 스토리 해금 | 합성 없음, **실시간 경제 압박(세금)**이 긴장감 원천 |
| Port Royale | 복잡한 무역·전투 시뮬레이션 | 진입장벽 낮음, **방치 허용** (배가 알아서 이동) |
| 트로피코 시리즈 | 도시 건설·정치 시스템 | 공간 관리 없음, **1~2분 세션**도 의미 있음 |

### 15.3 Pioneer의 고유 포지션

> **"방치하면서도 긴장감이 있는 무역 생존 게임"**

- **방치 허용**: 배가 항해하는 동안 앱을 꺼도 됨 → 바쁜 성인 유저 친화
- **세금 압박**: 단순 수집이 아니라 "버티기"가 목표 → 엔드게임 명확
- **짧은 세션**: 1~2분에도 의미 있는 행동 가능 → 이동 중 플레이 적합
- **경제 전략**: 가격 차트·정보 구매로 숙련도 차이 발생 → 재방문 동기

이 조합은 현재 모바일 시장에서 **직접적인 경쟁작이 없는 틈새**다.

## 16. 초보자 플레이 시나리오 — 0~30분

> **설계 목표**: 첫 30분 안에 플레이어가 "이 게임의 핵심 재미"를 체험하고 자발적으로 배를 다시 출발시키게 만든다.

### 16.1 분 단위 시나리오

```
[0~3분] 세계관 진입
  세계관 슬라이드 3장 (스킵 가능)
  → 시작 자금 100금 + 보석 5개 지급
  → 화면: 런던 항구 줌인, 첫 퀘스트 없음 (깨끗한 시작)

[3~7분] 첫 번째 배 구매 (tutorialStep 1)
  → 함선 탭 하이라이트: "슬루프를 구매하세요 (500금)"
  → 게임 시작 자금 100금 → 구매 불가?
  ★ 해결: 시작 자금을 600금으로 설정 (슬루프 500금 + 여유 100금)
  → 구매 완료: "슬루프 1호를 영입했습니다!" 로그 출력

[7~10분] 첫 선원 고용 (tutorialStep 2)
  → 선원 탭 하이라이트: "항해사 1명을 고용하세요 (50금)"
  → 일반 등급 항해사 1명 미리 배치 (50금 고용 유도)
  → 고용 완료: 선원 카드에 이름·효과 표시

[10~13분] 첫 항해 출발 (tutorialStep 3)
  → 지도 하이라이트: "슬루프를 클릭하고 목적지를 선택하세요"
  → 추천 목적지 배지 표시: "리스본 (15분)" — 가장 가까운 항구 강조
  → 출발 완료: 항로 골드 점선 애니메이션 시작
  → 팝업: "배가 출발했습니다! 앱을 꺼도 배는 항해합니다 ⚓"

[13~28분] 첫 방치 구간 (의도된 공백)
  → 타이머 배지: "슬루프 1호 — 리스본까지 12분"
  → 플레이어는 탭 구경, 가격 차트 확인 가능
  → 이 시간에 세금 타이머 미표시 (압박 없는 탐색 유도)

[28~30분] 첫 도착 + 첫 거래 (tutorialStep 4)
  → 로그 최상단: "★ 슬루프 1호가 리스본에 도착했습니다!"
  → 화물 탭 하이라이트: "자원을 구매해보세요"
  → 리스본에서 판매가 높은 자원 1종 강조 표시 (초보자 힌트)
  → 구매 완료 → 튜토리얼 종료 보상:
     💰 500금 + 💎 보석 2개 + 첫 퀘스트 자동 부여

[30분 이후] 자유 플레이
  → 플레이어는 이미 "출발 → 방치 → 도착 → 거래" 루프를 1회 완료
  → 첫 퀘스트: "긴급 배송 — 면화 10개를 런던으로" (쉬운 난이도)
  → 세금 첫 납부까지 약 20분 남음 (압박 시작 전 여유)
```

### 16.2 초반 이탈 방지 설계 원칙

| 위험 지점 | 이탈 원인 | 대응 설계 |
|-----------|-----------|-----------|
| 게임 시작 직후 | "뭘 해야 하는지 모름" | 튜토리얼 하이라이트 + 단계별 유도 |
| 첫 배 구매 시 | "돈이 부족함" | 시작 자금 600금으로 즉시 구매 가능 |
| 첫 항해 출발 후 | "기다리는 게 지루함" | 방치 의도 명시 팝업 + 타이머 배지 |
| 첫 도착 시 | "뭘 사야 할지 모름" | 고수익 자원 힌트 강조 표시 |
| 세금 첫 납부 | "갑자기 돈이 빠져나감" | 첫 납부 5분 전 경고 + 금액 미리 표시 |

### 16.3 첫 세션 성공 기준

플레이어가 30분 세션 종료 시점에 다음 상태여야 한다:

- [ ] 배 1척 보유, 항해 중 상태
- [ ] 선원 1명 배정 완료
- [ ] 금화 600금 이상 보유 (시작보다 증가)
- [ ] 퀘스트 1개 활성화 상태
- [ ] 세금 타이머 인지 완료 (다음 납부 시간 알고 있음)

이 5가지가 달성되면 플레이어는 **재방문 이유**가 생긴다.

## 17. 경제 밸런싱 심화

### 17.1 일별 성장 시뮬레이션

> 가정: 하루 3세션 (아침·점심·저녁), 세션당 약 5~10분, 세션마다 도착 화물 판매 + 재출발

| 일차 | 배 구성 | 세금 Lv | 일일 수익 | 일일 세금 | 일일 순수익 | 누적 자금 |
|------|---------|---------|----------|---------|-----------|---------|
| 1 | 슬루프 1 | L1 | ~900금 | 200금 | ~700금 | ~1,800금 |
| 3 | 슬루프 1 | L1 | ~900금 | 200금 | ~700금 | ~3,200금 |
| 5 | 슬루프 2 구매 후 | L2 | ~1,800금 | 600금 | ~1,200금 | ~2,600금 |
| 7 | 슬루프 2 | L2~3 | ~1,800금 | 600~1,000금 | ~1,000금 | ~5,000금 |
| 14 | 상인선 1 + 슬루프 1 | L3~4 | ~4,500금 | 1,000~3,000금 | ~2,500금 | ~17,000금 |
| 21 | 상인선 2 | L4~5 | ~6,000금 | 3,000~7,000금 | ~3,000금 | ~32,000금 |
| 30 | 갤리온 1 + 상인선 1 | L5~6 | ~9,000금 | 7,000~20,000금 | ~5,000금 | ~60,000금 |

**설계 확인**: 각 단계에서 일일 순수익이 양수를 유지해야 한다. 세금이 순수익을 초과하면 게임 오버 압박이 시작된다.

### 17.2 골드 흐름 분석 (Faucet vs Sink)

**수입원 (Faucet)**

| 수입원 | 비중 | 비고 |
|--------|------|------|
| 화물 판매 | ~75% | 주 수입, 세션마다 발생 |
| 퀘스트 보상 | ~15% | 세금 2~3배 보상, 핵심 동기 |
| 일일 목표 보상 | ~5% | 세션 유도 보조 수단 |
| 이벤트 보상 (고래 등) | ~3% | 랜덤, 소액 |
| 튜토리얼 완료 | ~2% | 1회성 500금 |

**지출원 (Sink)**

| 지출원 | 비중 | 비고 |
|--------|------|------|
| 세금 | ~35% | 게임 진행에 따라 비중 증가 |
| 배 구매/업그레이드 | ~40% | 최대 지출, 성장 투자 |
| 선원 고용 | ~15% | 성능 투자 |
| 정보 구매 | ~10% | 전략 투자 |

### 17.3 배 투자 ROI

| 배 추가 구매 | 비용 | 일일 추가 수익 | 세금 증가 | 순 회수 기간 |
|-------------|------|--------------|---------|------------|
| 2번째 슬루프 | 3,000금 | +900금/일 | +400금/일 | ~6일 |
| 상인선 (첫 구매) | 20,000금 | +2,800금/일 | +2,400금/일 | ~50일 → 장기 투자 |
| 갤리온 (첫 구매) | 45,000금 | +6,000금/일 | +13,000금/일 | 위험 — 수익 먼저 확보 필수 |

> **설계 원칙**: 배를 많이 살수록 세금이 급등하므로 무작정 확장이 최선이 아니다. 선원 투자와 루트 최적화가 배 추가보다 효율적인 구간이 반드시 존재해야 한다.

### 17.4 선원 투자 ROI

| 역할 | 등급 | 효과 | 일일 추가 수익 | 고용 비용 | 회수 기간 |
|------|------|------|--------------|---------|---------|
| 상인 | 일반 | 판매가 +7% | +63금/일 | 200금 | ~3일 |
| 상인 | 숙련 | 판매가 +14% | +126금/일 | 500금 | ~4일 |
| 상인 | 전문가 | 판매가 +22% | +198금/일 | 💎 1개 | 보석 희소성 고려 |
| 항해사 | 일반 | 속도 +8% | 왕복 시간 단축 | 150금 | 세션 빈도에 따라 다름 |

### 17.5 가격 변동 설계 범위

| 자원 등급 | 기준가 범위 | 변동폭 | 변동 주기 |
|----------|-----------|------|---------|
| 기초 (밀, 면화) | 50~150금 | ±20% | 1시간마다 |
| 중급 (향신료, 비단) | 200~500금 | ±30% | 1시간마다 |
| 고급 (보석류, 희귀품) | 800~2,000금 | ±40% | 1시간마다 |

**항구별 특산물 보정**: 해당 지역 특산물은 기준가 +30% ~ +80% 범위 내에서 시작하여 더 안정적인 수익을 제공한다.

## 18. UI/UX 설계

### 18.1 화면 구성 계층

```
App (전체 화면)
├── Header (고정, 상단)
│   ├── 타이틀 + 로고
│   ├── 💰 금화 표시
│   ├── 💎 보석 표시
│   ├── 🏛️ 세금 (다음 납부 카운트다운)
│   ├── ⚡ 시세 변동 배지 (클릭 시 제거)
│   └── 💾 저장 버튼
│
├── GameMap (메인 영역, 전체 높이)
│   ├── 지도 캔버스 (pan/zoom)
│   ├── 항구 마커 (지역별 이모지)
│   ├── 배 아이콘 + 이름 배지
│   └── 항로 골드 점선 애니메이션
│
└── Bottom Panel (탭 전환)
    ├── [함선] ShipPanel
    ├── [화물] TradePanel
    ├── [선원] CrewPanel
    ├── [정보] InfoPanel
    └── [퀘스트] QuestPanel
```

### 18.2 패널별 UX 상세

**GameMap**
- 배 클릭 → 해당 배 선택 + 하단 패널 [화물] 탭 자동 전환
- 항구 클릭 (항해 중이 아닌 배 선택 시) → 목적지 선택 확인 팝업
- 핀치 줌 / 마우스 휠 줌 지원
- 선택된 항구: `pulse-gold` 애니메이션 링

**ShipPanel**
- 배 목록: 카드 형태, 현재 상태 배지 (항해 중 / 항구 정박 / 항해 준비)
- 선택된 배: 상세 정보 (화물 현황, 선원 목록, 선체 내구도, 연료)
- [배 구매] 버튼: 배 목록 → 구매 확인 팝업 (비용 + 세금 레벨 상승 경고)
- [업그레이드] 버튼: 속도 / 화물 / 선원 슬롯 개선 (금화 소모)

**TradePanel**
- 자원 목록: 아이콘 + 이름 + 현재 가격 + 등락 표시 (▲▼)
- 자원 선택 → 하단에 수량 조정 슬라이더 (1 / 최대치 버튼 제공)
- [가격 차트] 탭: 클릭한 자원의 가격 히스토리 선 그래프
- 현재 화물 재고를 항상 상단에 표시

**CrewPanel**
- 역할별 필터 탭 (전체 / 항해사 / 상인 / 화물관리사 / 기관사)
- 선원 카드: 이름 + 역할 아이콘 + 등급 색상 테두리 + 능력치
- 배정 UI: 선원 카드 탭 → 배 선택 드롭다운
- [고용 가능] 탭: 항구에서 고용 가능한 선원 목록

**InfoPanel**
- 정보 목록: 등급별 (소문/귀띔/분석/내부정보)
- 각 항목: 비용 + 정확도 + 예상 효과 설명
- 이미 구매한 정보: 결과 표시 (예측이 맞았는지 여부)

### 18.3 알림 시스템

| 알림 종류 | 표시 위치 | 지속 시간 | 클릭 동작 |
|----------|---------|---------|---------|
| 배 도착 | 로그 최상단 + 헤더 배지 | 상시 | 해당 배 선택 |
| 시세 변동 | 헤더 ⚡ 배지 | 클릭 전까지 | 배지 제거 |
| 세금 경고 (4시간 전) | 헤더 세금 표시 빨강 전환 | 상시 | 세금 패널 열기 |
| 퀘스트 완료 | 로그 + 녹색 펄스 배지 | 클릭 전까지 | 퀘스트 패널 열기 |
| 이벤트 발생 | 화면 중앙 팝업 (EventOverlay) | 확인 전까지 | 회피 / 감수 선택 |

### 18.4 핵심 인터랙션 원칙

| 상황 | 패턴 | 이유 |
|------|------|------|
| 저렴한 항목 구매 (선원, 소량 화물) | 싱글 탭 확인 | 마찰 최소화 |
| 비싼 항목 구매 (배, 고급 선원) | 팝업 + 확인 버튼 | 실수 방지 |
| 대량 화물 거래 | 수량 슬라이더 + 총액 실시간 표시 | 계획적 거래 유도 |
| 세금 면제 / 즉시 도착 (보석 소모) | 팝업 + 보석 잔여량 표시 | 보석 가치 인지 |
| 게임 오버 위기 | 빨간 경고 UI + 해결 방법 제안 | 이탈 방지 |

### 18.5 모바일 터치 타겟

- 배 선택 터치 영역: 실제 아이콘보다 20px 확장 (작은 배 선택 용이)
- 슬라이더 트랙 높이: 최소 28px

## 19. 튜토리얼 가이드 체계

### 19.1 3단계 구성

```
Phase 1 — 필수 튜토리얼 (첫 30분)
  목표: 핵심 루프 1회 완성 + "이 게임 어떻게 하는지" 체득
  방식: 순차 하이라이트, 스킵 불가

Phase 2 — 맥락적 힌트 (Day 1~3)
  목표: 새 기능을 처음 마주칠 때 자연스럽게 설명
  방식: 첫 진입 시 툴팁 팝업, "다시 보지 않기" 옵션

Phase 3 — 심화 팁 (Day 3+)
  목표: 숙련 유저가 놓치기 쉬운 최적화 전략 제공
  방식: 선택적, 로그에 팁 노출, 도움말 패널 상시 열람 가능
```

### 19.2 Phase 1 — 필수 튜토리얼 단계

| Step | 하이라이트 대상 | 행동 요구 | 완료 조건 |
|------|---------------|---------|---------|
| 0 | 세계관 슬라이드 | 읽기 (스킵 가능) | 슬라이드 종료 |
| 1 | 함선 탭 | 슬루프 구매 | 구매 완료 |
| 2 | 선원 탭 | 선원 1명 고용 | 고용 완료 |
| 3 | 지도 → 목적지 | 배 출발 | 출발 완료 |
| 4 | 화물 탭 | 도착 후 자원 구매 | 구매 완료 |
| 완료 | — | — | 500금 + 💎2 + 첫 퀘스트 |

### 19.3 Phase 2 — 맥락적 힌트 트리거

| 트리거 조건 | 힌트 내용 | 표시 위치 |
|-----------|---------|---------|
| 첫 퀘스트 수락 | "퀘스트는 세금 2~3배 보상! 기한 안에 완료하세요" | 퀘스트 패널 상단 |
| 첫 이벤트 발생 | "회피: 보석 소모. 감수: 화물 손실. 항해사 선원이 있으면 확률 감소!" | 이벤트 팝업 하단 |
| 선원 2명 이상 보유 | "선원은 배에 배정해야 효과가 납니다. 선원 탭에서 배정하세요" | 선원 탭 첫 진입 시 |
| 금화 2,500금 달성 | "2번째 배를 사면 수익이 2배! 단, 세금도 오릅니다" | 로그 팁 |
| 세금 첫 납부 직전 (1시간) | "내일 세금 N금이 징수됩니다. 준비되셨나요?" | 헤더 경고 |
| 가격 차트 첫 오픈 | "차트에서 오르는 자원을 싸게 사두면 나중에 비싸게 팔 수 있어요" | 차트 패널 상단 |
| 선체 내구도 50% 이하 | "선체가 손상됐어요. 항구 도착 시 수리 가능합니다" | 로그 경고 |
| 일일 목표 첫 완료 | "일일 목표를 매일 완료하면 꾸준한 수익을 얻을 수 있어요" | 완료 팝업 |

### 19.4 Phase 3 — 심화 팁 (로그 노출)

| 조건 | 팁 내용 |
|------|--------|
| 세금 L3 도달 | "선원 상인 등급을 올리면 배를 추가하지 않아도 수익이 늘어요" |
| 배 3척 이상 운영 | "항구별 특산물 루트를 고정하면 시세 변동 리스크가 줄어요" |
| 정보 구매 첫 시도 | "대량 거래 전에 정보를 사면 수백 금 차이가 날 수 있어요" |
| 퀘스트 실패 | "퀘스트 실패 시 세금 1회분이 추가됩니다. 무리한 퀘스트는 거절하세요" |
| 보석 10개 이상 보유 | "보석은 세금 면제(2개)나 위기 이벤트 회피(1~3개)에 가장 효율적입니다" |

### 19.5 도움말 시스템 구조

```
[❓ 버튼] — 헤더 상시 노출
  └── 게임 요약: 핵심 루프 / 세금 시스템 / 선원 역할 / 보석 용도

[각 패널 (i) 아이콘]
  ├── ShipPanel: 배 종류별 스펙 비교표
  ├── TradePanel: 구매가/판매가 계산 공식 설명
  ├── CrewPanel: 역할별 효과 + 등급별 수치 범위
  ├── InfoPanel: 정확도 의미 + ROI 설명
  └── QuestPanel: 퀘스트 종류별 보상/패널티 설명

[세금 표시 탭]
  └── "세금이 왜 오르나요?" — 레벨 상승 조건 2가지 설명
```
