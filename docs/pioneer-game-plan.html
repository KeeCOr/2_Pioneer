게임 개요

대항해시대를 배경으로 한 무역 전략 게임. 리스본에서 출발해 유럽·지중해·아라비아·남아시아·동아시아 24개 항구를 탐험하며 상품을 사고팔아 부를 쌓는 것이 목표.

핵심 긴장감은 지수적으로 오르는 세금에 맞서 함대를 키우고 무역 루트를 최적화하며 얼마나 오래 살아남느냐에 있다.

3.2 능력치 계산식

속도     = baseSpeed × (1 + (항법평균-50)/200 + 속도업그레이드×0.15) × 연료보정
적재     = baseCapacity + 화물업그레이드×25
최대승무원 = min(14, t.maxCrew + 승무원업그레이드)
상술%    = round((거래평균-50)/2 × 선체보정)

연료보정: 연료 <30% → 0.5×  /  <60% → 0.75×  /  이상 → 1.0×
선체보정: 선체 <30% → 0.6×  /  <60% → 0.8×   /  이상 → 1.0×

3.3 업그레이드 (3종)

종류효과비용
속도 업그레이드항속 +15% / 레벨레벨 × 지수 증가
화물 업그레이드적재 +25개 / 레벨레벨 × 지수 증가
승무원 업그레이드최대 승무원 +1 / 레벨 (최대 14)레벨 × 지수 증가
ⓘ
업그레이드 패널에서 현재 수치 → 다음 레벨 수치 미리보기가 실시간 표시됨.

항구 시세 패널

항구를 클릭하면 오픈. 좌/우 분할 레이아웃 (v1.10 개선).

영역내용
헤더항구명 · 지역 뱃지 · 거래소 개장 여부 · 상승/하락 종수 · ✕ 버튼
좌측 목록 (144px)전체 상품 스크롤 목록 — 아이콘, 이름, 현재가, 등락률 뱃지. 클릭으로 우측 차트 선택
우측 차트선택 상품 대형 160px SVG 차트 + 현재가/등락 크게 표시 + 최고가·최저가·변동폭 3칸 + 수수료 안내
ⓘ
패널 열릴 때 첫 번째 상품 자동 선택. 오버레이 클릭 또는 ✕ 버튼으로 닫기. 차트는 순수 SVG(area + polyline + glow dot)로 구현.

시장 팝업 (매입/판매)

항구 정박 후 시장 버튼 클릭. 루트 레벨에 배치되어 맵 이벤트와 간섭 없음.

영역내용
헤더항구명 · 보유 금화 · ✕ 버튼
화물 바현재 적재량 / 최대 적재량 시각화
상품 목록상품별 구매 버튼 (1개 · 5개 · 10개)
하단연료 보충 · 선체 수리 버튼
⚠
시장 팝업은 맵 div 외부(루트 레벨 형제)에 위치. 내부에 있을 경우 stopPropagation이 setPointerCapture를 차단해 드래그 불가 버그 발생.

정보 시스템 (시세 예측)

8.1 정보 등급

등급이름기본 비용정확도변동폭반복
소문 거리 소문300금30%±15~45가능
힌트 상인 귀띔700금40%±25~65가능
분석 상업 분석 보고서3,000금58%±60~1301회

8.2 예측 흐름

구매 시 turnsUntil 결정 → 매 시세 변동마다 turnsRemaining-- → 0 도달 시 정확도 확률로 적용
소문: 1~3턴 / 힌트: 1~2턴 / 분석·내부: 1턴 (즉시)
최대 30개 보관 (오래된 것부터 삭제)

8.3 무료 소문 조건부 지급

시점조건내용
새 항구 최초 도착20% 확률소문 1개 자동 입수 (로그: 📰 XXX에서 시장 소문을 입수했습니다)
ⓘ
기존에 시세 변동·게임 시작 시 무료 제공하던 소문은 완전 폐지. 탐험 보상으로만 획득 가능하여 정보 구매의 가치가 높아짐.

퀘스트

9.1 일반 퀘스트 (3개 고정)

유형내용보상
운송특정 상품 N개를 특정 항구에서 판매금화
탐험새 항구 N곳 방문금화 + 보석 1
무역누적 판매액 N금 달성금화

퀘스트는 시세 변동 시마다 갱신 (3개 유지)

9.2 일일 목표 (4개, 매일 자정 리셋)

유형내용보상
일일 운송특정 상품 N개를 오늘 안에 판매금화 + 보석 1
일일 항구 순례새 항구 N곳 발견금화 + 보석 1
일일 매출 목표오늘 누적 판매액 N금금화
일일 거래 횟수오늘 판매 N회 완료금화
ⓘ
난이도는 세금 레벨에 따라 자동 스케일. 우측 상단 위젯 / 모달로 확인 가능.
10

세금 시스템

세금 공식
세금액 = 50 × 1.3^(level-1) + 선박수 × max(20, floor(20 × 1.1^(level-1)))
부과 주기: 매 600초

세금 레벨이 높을수록 퀘스트·일일 목표 난이도가 함께 상승. 이것이 게임의 핵심 긴장감을 만드는 생존 압박 메커니즘.

이벤트

12.1 해상 이벤트 (구현됨)

항해 중인 배에 5초마다 35% 확률로 이벤트 발생. 강제 이벤트가 대부분이며 즉각 적용.

이벤트유형가중치효과
⛈️ 폭풍우강제3545~90초간 속도 60% 감소
🏴‍☠️ 해적 출몰강제30화물 8~20% 즉시 약탈
🌊 순조로운 해류강제25연료 +8~18% 즉시 회복
🐋 고래 출몰강제20+80~200금 즉시 획득
🪵 난파선 발견클릭8클릭 시 +100~600금
💰 보물 발견클릭5클릭 시 +300~1,100금
⚠
강제 이벤트 비율 약 92% (가중치 합 110 중 110 × 92%). 클릭형 이벤트는 보조적 역할. 폭풍 지속 시간은 45~90초로 랜덤 변동.

12.2 날씨 시스템 (기획 완료, 구현 예정)

날씨속도 배율확률
잔잔1.0×50%
순풍1.3×20%
역풍0.7×20%
폭풍0.5×10%

폭풍 시 이동 중인 배 화물 최대 20% 손실 (스태미나 능력치로 완화).

18

성공 요인 분석

Pioneer가 캐주얼~중간 전략 게이머 시장에서 자리잡을 수 있는 핵심 이유를 분석.

18.1 게임플레이 설계

⏲
방치형 + 접속형의 균형
배를 출발시키고 현실 시간을 보내다 돌아와 결과를 처리하는 "출발 → 방치 → 귀환" 루프. 하루 5~15분 플레이로도 진행감을 느낄 수 있어 직장인·학생 코어 타겟에 최적.
▲
지수 세금 = 영원한 긴장감
세금이 지수적으로 오르기 때문에 "이기는" 엔드게임이 없다. 생존 시간 자체가 점수가 되어 리플레이어빌리티를 자연스럽게 확보. 소규모 전략 게임에서 가장 검증된 생존 압박 설계.
📈
정보 비대칭으로 전략 깊이 확보
시세 예측 정보 시스템(4등급)으로 "정보를 살 것인가, 위험을 감수할 것인가" 트레이드오프를 만든다. 단순 가격 비교 게임을 넘어 정보 투자 게임으로 격상.
★
희귀 승무원 수집 욕구
3% 확률의 전설 승무원은 강력한 가챠 동기를 만든다. 획득 시 즉각적인 성능 향상이 보장되어 유저 만족도 극대화. 수집 로그와 뱃지로 소장 가치도 부여.
📅
일일 목표의 재방문 유도
자정 리셋 일일 목표 4개는 매일 접속 이유를 제공. 세금 레벨에 따라 난이도가 오르므로 항상 적절한 도전감 유지. 보석 보상이 중장기 진행의 당근 역할.
📍
탐험의 만족감
24개 항구를 직접 발견하고, 각 지역의 상품 가격 구조를 학습하는 과정이 자연스러운 튜토리얼. "유럽에서 싸고 동아시아에서 비싸다"는 지식이 플레이어 역량으로 쌓이는 느낌을 준다.

18.2 기술·제품 품질

💾
설치 없는 포터블 exe
Electron 포터블 exe는 설치 마찰이 0. USB 하나로 어디서든 플레이 가능. 바이럴 공유에 최적이며, 기업 PC처럼 설치 권한이 없는 환경에서도 실행된다.
⚙
외부 의존성 최소화
차트를 외부 라이브러리 없이 순수 SVG로 구현, 상태 관리도 Redux 없이 useState + useRef만 사용. 번들 크기가 작고 로딩이 빠르며 장기 유지보수에 유리.
📷
화면 좌표 직접 렌더링 (블러 없는 줌)
CSS scale 레이어 대신 스크린 좌표로 직접 렌더링하는 방식으로 줌 시 픽셀 블러가 없다. 사용자가 의식하지 못해도 체감 품질에 기여하는 디테일.
🔙
자동 저장으로 데이터 손실 0
매 시세 변동마다 localStorage에 자동 저장. 게임이 갑자기 꺼져도 진행이 날아가지 않는다는 신뢰감은 장기 플레이어 유지에 핵심.

18.3 시장 포지션

경쟁 우위 요약
관점Pioneer의 강점
장르 레퍼런스 대항해시대 IP 감성 + 현대적 UI/UX. 원작 팬층과 전략 게임 신규 유저 모두 공략.
진입 장벽 다운로드 → 실행으로 즉시 플레이. 튜토리얼 없이도 직관적 구조.
세션 길이 5분~무제한. 캐주얼과 하드코어 모두 수용하는 유연한 플레이 타임.
수익화 잠재력 보석 5종 시스템으로 페이-투-윈 없이 QoL 중심 과금. 유저 반감 최소화.
개발 효율 단일 파일 → 모듈 분리 경로 확보. 현재 코드베이스가 v2.0 리팩토링 기반이 됨.

18.4 개선 우선순위 (성공 확률 극대화)

순위기능이유
1순위 FTUE 인터랙티브 튜토리얼 첫 10분 경험이 리텐션을 결정. 지금은 자력으로 배워야 함.
2순위 날씨·해적 이벤트 단조로운 A→B 이동에 위기감과 선택지 추가. 몰입도 급상승.
3순위 v2.0 모듈 분리 이후 기능 추가 속도를 4배 이상 높임. 장기 성공의 기반.
4순위 리더보드 / 생존 기록 "내 기록 몇 레벨?"을 공유하는 소셜 루프. 바이럴 확산 엔진.
5순위 보석 수익화 안정적 유저 기반 확보 후 도입. 성급하면 역효과.
